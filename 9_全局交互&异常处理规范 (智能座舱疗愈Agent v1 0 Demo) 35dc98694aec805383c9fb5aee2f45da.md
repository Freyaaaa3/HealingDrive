# 9_全局交互&异常处理规范 (智能座舱疗愈Agent v1.0 Demo)

**模块版本**：v1.0 Demo
**文档状态**：正式PRD
**更新日期**：2026-05-11

## 一、模块概述

本规范为智能座舱疗愈Agent所有模块**统一交互标准、全局规则、通用异常兜底策略**。
适用于唤醒、形象、语音、情绪识别、疗愈服务、画像、数据存储、车企配置全链路，统一约束交互范式、时效标准、容错降级、异常提示原则，保证整车体验一致、稳定、不打扰驾驶。

## 二、全局通用交互规范

### 2.1 核心交互原则

1. **全程纯语音交互**，无任何触控点击、无界面按钮，杜绝驾驶分心。
2. **零弹窗、无Toast、无文字提示**，所有反馈仅靠语音+2D形象动效。
3. 情绪识别+首次完整回应**严格≤2秒**。
4. 后台低功耗运行，不抢占座舱核心资源、不干扰导航/通话/音乐。
5. 所有失败/异常**静默降级**，不弹错误窗口、不打断驾驶。

### 2.2 语音通用规则

| 需求点 | 规范规则 |
| --- | --- |
| 语言限制 | Demo仅标准普通话，不支持方言、外语 |
| 音色统一 | 固定治愈温柔女声，全局一致 |
| 语速语调 | 平缓低沉、不急促、不高亢，适配车载场景 |
| 音频优先级 | 导航/蓝牙通话 ＞ 疗愈语音 ＞ 车载音乐 |
| 连续对话 | 唤醒后30秒免唤醒自然对话 |
| 自动退出 | 30秒无交互自动收起形象、停止倾听 |

### 2.3 2D形象通用规范

| 需求点 | 规范规则 |
| --- | --- |
| 展示层级 | 悬浮层，不遮挡仪表盘、导航车道、限速等行车关键信息 |
| 动效状态 | 待机/倾听/说话三种固定姿态，低功耗设计 |
| 无触控 | 形象不可点击、不可拖拽，完全禁止手动交互 |
| 异常兜底 | 资源加载失败自动隐藏形象，仅保留语音正常交互 |

### 2.4 权限通用规则

1. 必备权限：麦克风、摄像头、地图
2. 权限缺失**静默降级**，不弹窗申请、不提示报错
3. 无麦克风：直接关闭全部疗愈能力
4. 无摄像头：情绪识别自动剔除面部维度
5. 无地图：关闭停车点推荐能力

### 2.5 数据通用规则

1. 所有情绪/画像/记录**仅本地存储，不上云不上报**
2. 不保存原始语音、人脸原图、精准位置等敏感隐私数据
3. 数据异常不弹窗，自动用默认配置兜底
4. 支持语音一键清空所有本地记录并重置画像

## 三、全局统一降级策略

### 3.1 硬件缺失降级

- 无麦克风：完全无法唤醒，后台静默监听
- 无摄像头：情绪仅用语音+驾驶行为+画像识别
- 无地图：关闭停车推荐，其余能力正常
- 座舱性能不足：关闭形象动效，保留静态+语音

### 3.2 服务异常降级

- 大模型响应超时：自动调用本地预设安抚话术
- ASR识别失败：通用温柔兜底回复
- TTS播报异常：仅形象动效，不影响对话逻辑
- 音乐资源失败：语音温和提示无法播放
- 正念语音异常：可重新语音发起

### 3.3 场景冲突降级

- 导航播报中：自动降低疗愈语音音量
- 蓝牙通话中：暂停所有倾听与播报
- 座舱音频被占用：拦截唤醒，静默待命
- 系统弹窗覆盖：形象自动后置层级

## 四、全局统一超时规则

| 超时场景 | 标准时长 | 处理逻辑 |
| --- | --- | --- |
| 情绪识别+首次回应 | ≤2s | 超时仍正常回复，不提示 |
| 免连续对话时长 | 30s | 无交互自动收起形象、关闭倾听 |
| 语音指令等待 | 30s | 未响应自动失效 |
| 座舱后台休眠 | 自动暂停 | 唤醒后自动恢复监听 |

## 五、全局禁止行为（硬性规范）

1. **禁止任何弹窗报错、禁止文字Toast提示**
2. **禁止驾驶中弹出遮挡行车关键UI**
3. **禁止采集上传任何隐私敏感数据到云端**
4. **禁止用户手动切换形象/音色/系统配置**（Demo车企固化）
5. **禁止复杂操作、禁止需要视觉聚焦的交互**
6. **禁止高频率动效、高GPU占用渲染**

## 六、全模块通用异常处理汇总

### 6.1 语音唤醒&交互类

- 唤醒词识别失败：静默无响应
- 麦克风权限禁用：无法唤醒，后台静默
- 环境噪音过大：语音提示“没听清，请再说一遍”
- ASR识别失败：触发通用安抚兜底话术
- TTS加载异常：仅形象不动效，语音静默兜底

### 6.2 虚拟形象类

- 形象资源加载失败：隐藏形象，语音正常
- 动效卡顿：自动降级为静态形象
- 屏幕适配异常：自动等比缩放、避让挖孔
- 层级错乱：自动后置不遮挡行车信息

### 6.3 情绪识别类

- 多维度数据部分缺失：自动重新加权计算
- 识别超时>2s：不弹窗，正常输出结果
- 情绪特征模糊：默认降级为轻度焦虑
- 初次无用户画像：使用通用基准模型

### 6.4 疗愈服务类

- 大模型超时：本地固定安抚话术兜底
- 音乐播放失败：语音温和提示暂时无法播放
- 正念引导中断：可重新语音发起
- 停车点无数据：语音友好提示暂无推荐

### 6.5 画像&数据存储类

- 画像计算异常：保留历史稳定标签
- 本地存储损坏：重置默认画像
- 单次数据异常：自动过滤不纳入统计
- 清空指令失败：不弹窗，保持原数据

### 6.6 车企配置类

- 配置文件缺失：自动回落Demo默认配置
- 形象资源替换异常：使用默认占位
- 音色加载失败：兜底治愈女声
- 超时配置非法：强制固定30s兜底

## 七、全局设计约束

1. 所有文案、语音话术**必须适配车载驾驶氛围**，不情绪化、不说教、不亢奋。
2. 所有异常只做**静默降级+极简语音提示**，绝不干扰司机驾驶。
3. 所有能力设计遵循「先保证驾驶安全，再提供疗愈体验」。
4. Demo版所有容错、降级、异常逻辑全部闭环，无需额外开发适配。
